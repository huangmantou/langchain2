# 服装客服智能Agent开发准则

本项目是基于 LangChain ReAct Agent 的服装智能客服系统（参考优衣库风格）。
在修改、扩展本项目时，必须严格遵守以下原则：

## 十二荣耻

1. 以瞎猜接口为耻，以认真查询为荣
2. 以模糊执行为耻，以寻求确认为荣
3. 以臆想业务为耻，以人类确认为荣
4. 以创造接口为耻，以复用现有为荣
5. 以跳过验证为耻，以主动测试为荣
6. 以破坏架构为耻，以遵循规范为荣
7. 以假装理解为耻，以诚实无知为荣
8. 以盲目修改为耻，以谨慎重构为荣

## 具体执行要求

- **不瞎猜**：修改代码前必须先完整阅读相关文件，确认现有结构后再动手。
- **不臆想业务**：服装领域的产品知识、尺码规则、面料特性、退换货政策等不确定的内容，必须先向用户确认，不得自行编造。
- **复用现有**：RAG 流程、中间件机制、配置加载、工具函数等现有架构全部复用，只改内容层。
- **主动验证**：修改完成后检查文件一致性，确保 import、配置路径、工具名、提示词引用都对得上。
- **遵循规范**：保持原有代码风格（命名规范、注释风格、目录结构、YAML 配置方式）。
- **诚实无知**：不确定的优衣库/服装业务细节主动向用户提问，不自行脑补。
- **谨慎重构**：除非用户明确要求，不对现有架构做重构。修改范围严格限定在任务所需。
- **认真查询**：涉及工具函数、配置项、文件路径时，必须先读取实际文件内容确认，不凭记忆假设。
- **寻求确认**：遇到多种实现方式或业务理解歧义时，先列出选项让用户选择。
- **主动测试**：修改完成后主动检查代码逻辑一致性，必要时运行验证。
- **不创造接口**：不新增不必要的工具函数或配置项，优先使用已有能力。
- **不模糊执行**：每一步操作都要明确目的和预期结果，不做含糊的批量修改。
